# go.md

## 基础
- 遵循 `gofmt`、Go 官方命名与现有 package 划分。
- 包名小写短词，按职责划分，如 `service`、`repo`、`api`、`model`。
- 保持现有目录结构、接口定义、错误语义稳定。
- 包、结构体、导出函数必须有注释，说明职责与用法。

## 设计原则
- 单一职责：结构体、函数、方法只负责一项核心职责；复杂逻辑按数据处理、业务逻辑、IO 分离；函数尽量 50 行内。
- 开闭原则：新增能力优先通过接口、组合、配置扩展，不直接修改稳定核心逻辑兼容新场景。
- 里氏替换：实现必须兼容接口约束，入参、返回值、错误规则保持一致，可无缝替换。
- 接口隔离：接口小而精，建议 1~3 个强相关方法，禁止要求实现无关方法。
- 依赖倒置：依赖抽象接口，通过 DI 传递依赖，禁止在业务函数内硬编码实例化具体依赖。
- 迪米特法则：只与直接依赖通信，最小化暴露内部字段/方法，减少无关第三方包依赖。
- 组合复用：优先组合/聚合，避免深层嵌套，组合层级不超过 2 层。

## 强制约束
- 必须显式处理并返回 `error`，不隐藏、不忽略错误。
- 核心函数必须编写单测，基于接口依赖实现 mock。
- 禁止新增全局变量；配置与依赖通过依赖注入传递。

## 检查
- 有条件执行 `gofmt`、`go vet ./...`、`go build ./...`。
