# common.md

## 工作边界
- 仅在当前 workspace 工作；不下载远程代码。
- 改前先看上下文、调用链、相似实现；支持并发读取/搜索。
- 规则冲突时取更保守、更兼容、更小改动方案。
- 发现架构问题先简述问题、建议、收益、风险，获同意再改。

## 修改原则
- 最小必要改动；只改任务相关文件；不做无关重构。
- 优先复用现有实现并保持行为稳定。
- 不擅改公共接口、对外字段、错误语义、响应结构。
- 不擅自新增第三方依赖；非必要不使用第三方组件，必须使用时先确认。
- 新增/修改方法、函数、关键变量必须补注释，说明职责、边界、关键约束。
- 不在仓库保留缓存、日志、构建产物、临时文件。

## 默认流程
1. 读任务与规则
2. 查上下文、调用链、相似实现
3. 按最小改动实施
4. 自检并尽量执行编译、测试、lint
5. 真实说明改动、原因、影响、验证结果

## 输出格式
- 每次任务完成后固定输出 1~7 项：修改文件、改动内容、改动原因、准确性检查、潜在影响、格式化/编译/测试说明、后续优化建议。
- 写真实文件路径；无改动写“未修改文件”。
- 只列实际执行的检查与结果；未执行需说明原因，不得把理论可行说成已验证通过。
