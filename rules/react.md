# react.md

## React
- 默认使用类组件；仅非常简单、无复杂业务状态组件可用函数组件
- 保持现有组件结构、路由、状态流、接口调用方式一致
- 不为“更优雅”改变业务逻辑、props 契约、接口字段或页面交互
- 先修正确性，再做结构和性能优化
- 类组件中复杂 JSX 拆分到 `renderXxx()` 方法
- 组件与业务逻辑文件统一使用 `.jsx` 后缀
- 优先使用 React 原生 API；仅在必要时引入第三方库

## 约束
- TypeScript 代码保持明确类型
- 不擅自新增状态管理库
- 请求层、签名、鉴权、错误处理改动必须与后端行为对齐

## 检查
- 有条件优先执行 `npm run build`
- 有条件优先执行 `npm run lint`
